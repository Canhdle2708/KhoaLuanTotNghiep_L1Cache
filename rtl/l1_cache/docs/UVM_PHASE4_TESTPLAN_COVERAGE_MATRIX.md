## Evidence Policy

- `PASS` is used only when current UVM monitor, scoreboard, perf counter, or verified test log evidence directly supports the testcase intent.
- `PARTIAL` is used when observable UVM evidence covers part of the testcase, but a deep/internal condition or exact scenario is not proven.
- `DEFERRED` is used for deep PLRU/VBUF/MSHR/RTAB/arbiter internals or stress/performance/random infrastructure that is not yet reliable in UVM Phase 4.
- `NOT_COVERED` is used when no current UVM evidence covers the testcase.
- Performance-sheet-only checks are not imported into this matrix; stress rows that already exist in `TestCase` remain represented but are deferred to the later stress/performance phases.
