# UVM Phase 2 File Change Plan

This is a proposal only. No Phase 2 code is implemented by this report.

| Action | File Path | Reason | Risk | Notes |
|---|---|---|---|---|
| CREATE | `rtl/l1_cache/work/uvm/txn/cv32_l1_sys_txn.sv` | System events and final status transaction | Low | Add compile-only first. |
| CREATE | `rtl/l1_cache/work/uvm/txn/cv32_l1_core_txn.sv` | Instruction/data port transaction | Medium | Store response semantics need confirmation. |
| CREATE | `rtl/l1_cache/work/uvm/txn/cv32_l1_mem_txn.sv` | I-cache and D-cache memory transaction | Medium | Split write protocol needs careful modeling. |
| CREATE | `rtl/l1_cache/work/uvm/txn/cv32_l1_event_txn.sv` | Basic cache event transaction | Low | Use for miss/refill/request events only. |
| MODIFY | `rtl/l1_cache/work/sim/cv32_l1_uvm_questa.f` | Explicitly compile new txn files | Low | Do not use wildcard `*.sv`. |
| MODIFY | `rtl/l1_cache/work/uvm/pkg/cv32_l1_uvm_pkg.sv` | Include txn classes before env/monitor classes | Low | Keep include order explicit. |
| MODIFY | `rtl/l1_cache/work/uvm/env/cv32_l1_env.sv` | Connect monitor analysis ports to scoreboard/perf collector | Medium | Connect one monitor at a time. |
| MODIFY | `rtl/l1_cache/work/uvm/env/cv32_l1_scoreboard.sv` | Add transaction sinks and basic pending/count checks | Medium | Start ambiguous checks as warnings. |
| MODIFY | `rtl/l1_cache/work/uvm/env/cv32_l1_perf_collector.sv` | Count txns and write Phase 2 CSV | Low | Keep existing `uvm_perf_raw.csv`. |
| MODIFY | `rtl/l1_cache/work/uvm/monitors/cv32_l1_sys_monitor.sv` | Emit `cv32_l1_sys_txn` | Low | Already performs X/Z check. |
| MODIFY | `rtl/l1_cache/work/uvm/monitors/cv32_l1_core_monitor.sv` | Emit core instruction/data txns | Medium | Response matching is the main risk. |
| MODIFY | `rtl/l1_cache/work/uvm/monitors/cv32_l1_mem_monitor.sv` | Emit memory read/write txns | Medium | Current mem interface should be extended. |
| MODIFY | `rtl/l1_cache/work/uvm/monitors/cv32_l1_icache_monitor.sv` | Emit event txns for miss/refill | Low | Do not add deep I-cache checker yet. |
| MODIFY | `rtl/l1_cache/work/uvm/monitors/cv32_l1_dcache_monitor.sv` | Emit event txns for data/cache events | Low | Do not classify complex miss causes yet. |
| MODIFY | `rtl/l1_cache/work/uvm/if/cv32_l1_mem_if.sv` | Add explicit split read/write protocol fields if needed | Medium | Existing names are incomplete and mixed. |
| MODIFY | `rtl/l1_cache/work/uvm/tb/tb_cv32_l1_uvm_top.sv` | Connect new mem_if fields if added | Medium | Keep Phase 1 signal mappings intact. |
| KEEP_REFERENCE | `rtl/l1_cache/work/tb/tb_cv32e40p_full_l1_basic.sv` | Golden non-UVM basic behavior reference | Low | Do not modify. |
| KEEP_REFERENCE | `rtl/l1_cache/work/rtl/cv32e40p_full_l1_cache_top.sv` | Source of confirmed signal paths/counters | Medium | Do not modify unless later absolutely necessary. |
| KEEP_REFERENCE | `rtl/l1_cache/docs/UVM_PHASE1_QUESTA_RESULT.md` | Baseline proof | Low | Do not overwrite except by baseline runner. |
| KEEP_REFERENCE | `rtl/l1_cache/logs/uvm_full_l1_basic_questa.log` | Baseline log | Low | New runs may overwrite internal runner log. User tee logs should be preserved separately. |
| DO_NOT_TOUCH | HPDCache base RTL source tree | Keep source baseline intact | High | No RTL edits for Phase 2 proposal. |
| DO_NOT_TOUCH | HPDCache patched/head filelists or source trees | Avoid mixing patched implementation | High | Phase 2 must remain base mode. |
| DO_NOT_TOUCH | CV32E40P RTL source files | Avoid unrelated RTL changes | High | Use UVM wrappers/interfaces instead. |
| DO_NOT_TOUCH | CVA6 I-cache RTL source files | Avoid unrelated RTL changes | High | Use visible top/interface signals. |
| DO_NOT_TOUCH | Legacy functional coverage scripts/testbenches | Preserve previous work | High | No move/delete/overwrite. |
| DO_NOT_TOUCH | `run_uvm_questa.sh` default coverage behavior | Preserve Questa Starter compatibility | Medium | Keep `QUESTA_ENABLE_COVERAGE=0` default. |
| CREATE | `rtl/l1_cache/docs/UVM_PHASE2_RESULT.md` | Record implementation results after later Phase 2 work | Low | Create only after implementation and rerun. |
| CREATE | `rtl/l1_cache/logs/uvm_phase2_transaction_counts.csv` | Transaction count output | Low | Generated by later implementation, not this proposal. |

Recommended implementation gate sequence:

1. Add txn files and filelist/package includes only.
2. Rerun `make HPDCACHE_SRC_MODE=base run_uvm_full_l1_basic`.
3. Add sys monitor analysis port and scoreboard sink.
4. Rerun baseline.
5. Add core monitor txns.
6. Rerun baseline.
7. Add memory monitor txns.
8. Rerun baseline.
9. Add CSV output.
10. Rerun baseline and create Phase 2 result doc.
