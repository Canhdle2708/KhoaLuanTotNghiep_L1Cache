## Common File Changes

`cv32_l1_uvm_pkg.sv`

- Added explicit includes for Phase 3 utilities and tests.
- Risk: low; include order keeps base seq/base test before Phase 3 extensions.

`cv32_l1_uvm_questa.f`

- Added `+incdir+../uvm/phase3_directed`.
- Risk: low; no wildcard source inclusion added.

`run_uvm_questa.sh`

- Made UVM test selection parametric through `UVM_TESTNAME`.
- Added `PASS_MARKER`, `FAIL_MARKER`, `UVM_LOG_BASENAME`, `BENCH_NAME`, and `WRITE_RESULT_DOCS`.
- Replaced hardcoded license path with a relative candidate path.
- Redirected Questa transcript output to `rtl/l1_cache/logs` through `vsim -l`.
- Risk: medium; shared runner is used by Phase 1/2 and Phase 3. Defaults preserve `run_uvm_full_l1_basic`.

`run_compare_3mode_perf.sh`

- Changed `DCACHE_ONLY_ROOT` fallback from a hardcoded `/media` path to a relative sibling repo lookup.
- Risk: low; users can still override `DCACHE_ONLY_ROOT`.

`hpdcache_src_mode.sh`

- Generated HPDCache filelist entries now use paths relative to `SIM_DIR` when `realpath` is available.
- Blank lines and `//` comment lines are skipped when generating simulator filelists, so Questa does not parse comments as malformed path arguments.
- Risk: medium; affects generated source filelists, but simulator commands run from `work/sim`, so relative paths are expected to resolve correctly.

`Makefile`

- Added Phase 3 individual targets and suite target.
- Changed `DCACHE_ONLY_ROOT` default from a hardcoded `/media` path to a relative computed path.
- Risk: low for Phase 3; performance compare users can still override `DCACHE_ONLY_ROOT`.

Generated HPDCache filelists

- Rewrote old absolute prefixes in existing `hpdcache_*.f` generated filelists to relative paths.
- Risk: low for base UVM Phase 3; these filelists are regenerated by `hpdcache_src_mode.sh`, which now emits relative paths.
